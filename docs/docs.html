Introduction

Darbot Graph MCP Server is an ultimate extensible MCP (Model Context Protocol) server for Microsoft Graph API. It bridges AI assistants with Microsoft 365 services through 64+ comprehensive tools built on official Microsoft Graph SDKs.

💡

What is MCP?

Model Context Protocol (MCP) is a standard for connecting AI assistants to external tools and services. It enables AI models to interact with real-world APIs and data sources.

Key Features

  • 64+ Comprehensive Tools: Coverage of Microsoft Graph API operations
  • Official SDK Foundation: Built on Microsoft.Graph SDK v5.x and Beta SDK
  • Hierarchical Organization: Optimized for VS Code's tool limits
  • Dual SDK Support: Access to both stable and preview APIs
  • Demo Mode: Safe testing without production data
  • Enterprise Security: Azure AD authentication with best practices

Architecture

The server is built using .NET 8.0 and follows a modular architecture:

  • MCP Server Layer: Handles protocol communication
  • Service Layer: Microsoft Graph API integration
  • Tool Categories: Organized by functional domain
  • Authentication Layer: Azure AD integration

Installation

Option 1: One-Click Installation (Recommended)

The fastest way to get started:

  1. Click the install button for your VS Code version:
Install for VS Code Install for VS Code Insiders
  1. Enter your Azure AD credentials when prompted (or leave empty for demo mode)
  2. Wait for installation to complete
  3. Restart VS Code

Option 2: NPX Installation

npx -y @darbotlabs/darbot-graph-mcp [TENANT_ID] [CLIENT_ID] [CLIENT_SECRET]

Option 3: Manual Installation

git clone https://github.com/dayour/darbot-graph-mcp.git cd darbot-graph-mcp dotnet restore dotnet build dotnet run --project src/DarbotGraphMcp.Server

Quick Start

1. Verify Installation

After installation, verify the server is running by checking the MCP status in VS Code.

2. Test with Demo Mode

If you installed without Azure AD credentials, you're in demo mode. Try these commands:

darbot-graph-users-list darbot-graph-mail-send darbot-graph-teams-list

3. Configure Azure AD (Production)

For production use, configure Azure AD authentication. See Azure AD Setup.

⚠️

Important

Demo mode provides simulated responses only. For actual Microsoft 365 operations, you must configure Azure AD authentication.

Prerequisites

Required Software

  • VS Code or VS Code Insiders with MCP extension
  • .NET 8.0 SDK or later (Download)
  • Node.js 16+ for NPX installation

Azure Requirements (Production)

  • Azure subscription with admin access
  • Azure Active Directory tenant
  • Microsoft 365 tenant (optional, for M365 operations)
  • Global Administrator or Application Administrator role

Permissions

The following Microsoft Graph API permissions are required:

User.ReadWrite.All Group.ReadWrite.All Mail.ReadWrite Calendars.ReadWrite Files.ReadWrite.All Sites.ReadWrite.All Team.ReadBasic.All Reports.Read.All SecurityEvents.Read.All AuditLog.Read.All

Azure AD App Registration

Step 1: Register Application

  1. Sign in to the Azure Portal
  2. Navigate to Azure Active Directory → App registrations
  3. Click New registration
  4. Enter application details:
    • Name: Darbot Graph MCP
    • Supported account types: Single tenant
    • Redirect URI: Leave empty
  5. Click Register
✅

Success

Your application is now registered. Note the Application (client) ID and Directory (tenant) ID from the overview page.

Step 2: Configure API Permissions

  1. In your app registration, go to API permissions
  2. Click Add a permission → Microsoft Graph
  3. Select Application permissions
  4. Add the following permissions:
    • User.ReadWrite.All
    • Group.ReadWrite.All
    • Mail.ReadWrite
    • Calendars.ReadWrite
    • Files.ReadWrite.All
    • Sites.ReadWrite.All
    • Team.ReadBasic.All
    • Reports.Read.All
    • SecurityEvents.Read.All
    • AuditLog.Read.All
  5. Click Grant admin consent
⚠️

Admin Consent Required

Application permissions require admin consent. Ensure you have Global Administrator or Application Administrator role.

Client Secrets

Create Client Secret

  1. In your app registration, go to Certificates & secrets
  2. Click New client secret
  3. Enter description: Darbot Graph MCP Secret
  4. Select expiration: 24 months (recommended)
  5. Click Add
  6. Copy the secret value immediately (you won't be able to see it again)
⚠️

Security Warning

Never commit client secrets to source control. Store them securely in environment variables or Azure Key Vault.

Secret Rotation

Regularly rotate client secrets for security:

  • Set calendar reminder before expiration
  • Create new secret before old one expires
  • Update configuration with new secret
  • Verify functionality
  • Delete old secret

VS Code Configuration

MCP Server Configuration

Add the following to your VS Code MCP configuration:

{ "mcpServers": { "darbot-graph": { "command": "dotnet", "args": [ "run", "--project", "/path/to/darbot-graph-mcp/src/DarbotGraphMcp.Server" ], "env": { "AzureAd__TenantId": "your-tenant-id", "AzureAd__ClientId": "your-client-id", "AzureAd__ClientSecret": "your-client-secret" } } } }

Configuration Options

Option Description Required
AzureAd__TenantId Azure AD tenant ID No (demo mode if empty)
AzureAd__ClientId Application (client) ID No (demo mode if empty)
AzureAd__ClientSecret Client secret value No (demo mode if empty)

Demo Mode

Overview

Demo mode allows you to test Darbot Graph MCP without Azure AD credentials. It provides simulated responses for all tools.

Enabling Demo Mode

Simply leave Azure AD credentials empty during installation or in configuration.

Simulated Responses

Demo mode returns realistic sample data for:

  • User lists and details
  • Group information
  • Email operations
  • Calendar events
  • Teams and channels
  • File operations
  • Security reports
💡

Use Cases for Demo Mode

  • Learning the tool structure
  • Testing AI assistant integrations
  • Development and debugging
  • Demonstrations and presentations

Security Best Practices

Authentication

  • Use app-only authentication for automation scenarios
  • Implement delegated authentication for user-specific operations
  • Enable multi-factor authentication (MFA) for admin accounts
  • Use managed identities when running in Azure

Permissions

  • Follow principle of least privilege
  • Request only required permissions
  • Regularly audit granted permissions
  • Use application permissions for background tasks

Secrets Management

  • Never commit secrets to source control
  • Use Azure Key Vault for production
  • Rotate secrets regularly (every 90 days recommended)
  • Use environment variables for local development

Network Security

  • Use HTTPS for all communications
  • Implement IP restrictions if possible
  • Enable conditional access policies
  • Monitor and log all API calls

For complete security guidelines, see the Security Guide.

Troubleshooting

Common Issues

Authentication Errors

Error: AADSTS700016: Application not found
Solution: Verify Client ID is correct and app is registered in the correct tenant.
Error: AADSTS7000215: Invalid client secret
Solution: Client secret may have expired or is incorrect. Create a new secret.

Permission Errors

Error: Insufficient privileges
Solution: Ensure required API permissions are granted and admin consent is provided.

Installation Issues

Issue: Tools not showing in VS Code
Solution: Completely restart VS Code (close all windows) and reopen.
Issue: Build errors during installation
Solution: Verify .NET 8.0 SDK is installed: dotnet --version

For more troubleshooting steps, see the Troubleshooting Guide.
