Extensibility

Extend Darbot Graph MCP with custom tools

Architecture Overview

Darbot Graph MCP is designed for maximum extensibility to accommodate the full breadth of Microsoft Graph APIs.

Modular Design

  • Tool Categories: Organized by functional domain
  • Service Layer: Abstraction for Graph API calls
  • SDK Support: Both v1.0 and Beta APIs
  • Naming Convention: Hierarchical tool naming

Adding New Tools

Step 1: Identify Graph API

Choose from the Microsoft Graph API reference.

Step 2: Design Tool Schema

Define input parameters and validation rules.

// Example tool schema { "name": "darbot-graph-{category}-{action}", "description": "Tool description", "inputSchema": { "type": "object", "properties": { "param1": { "type": "string", "description": "..." } }, "required": ["param1"] } }

Step 3: Implement Service Method

Add async method in GraphServiceEnhanced.cs

public async Task<string> MyCustomOperationAsync(string param1) { // Use _graphClient for stable API // Use _betaGraphClient for preview API var result = await _graphClient .Users .GetAsync(); return JsonSerializer.Serialize(result); }

Step 4: Add Tool Definition

Include in appropriate ToolCategories method.

new Tool { Name = "darbot-graph-{category}-{action}", Description = "Tool description", InputSchema = new { type = "object", properties = new { ... } } }

Step 5: Test & Document

  • Test in demo mode
  • Test with real Azure AD
  • Update cmd_lib.md
  • Add examples

Supported Graph API Areas

Ready for Implementation 🔄

  • Device Management: Intune device operations, compliance policies
  • Identity Protection: Conditional access, risk policies
  • Directory Management: Administrative units, organizational contacts
  • Contacts: Personal and organization contact management
  • Tasks/Planner: Microsoft Planner integration
  • OneNote: Notebook, section, and page operations
  • Licenses: Subscription and license management
  • External Identities: B2B collaboration and B2C
  • Bookings: Microsoft Bookings appointment management
  • Education: Education-specific classes and assignments
  • Compliance: Data loss prevention, retention policies
  • Search: Microsoft Search query and administration
  • Cloud Communications: Teams calls and meetings
  • Print: Universal Print service management

SDK Version Management

Stable Operations

Use _graphClient for production-ready APIs (Microsoft.Graph v5.x).

Preview Features

Use _betaGraphClient for beta/preview APIs (Microsoft.Graph.Beta).

⚠️

Beta API Caution

Beta APIs may change without notice. Use stable APIs for production scenarios.
